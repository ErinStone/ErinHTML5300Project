6/8/2015 - Annotate Images

I created this sample HTML and javascript page to demonstrate how you can use a canvas element to capture edits to an image. I needed to do this sample for a larger project I was working on. In this sample, I use the Signature Pad javascript library to create the edits. I used this particular library because it also works on mobile devices and why re-invent the wheel? After you create your edit, you can elect to save it and the application *should* merge both canvas elements into one creating a new image all-together.

5/31/2015 - Gimme Ice Cream

I have added in a link to a basic web application developed using Microsoft ASP.NET MVC4 framework. This is an application my co-worker John McIntyre and I created to handle our group's monthly ice-cream ordering. The front end uses ASP.NET MVC4 for layout/interaction. This app lives in the Microsoft Azure cloud (hence the azurewebsites.net in the name). The backend uses Azure Storeage to store user information and past order history. To login and get a feel for it, you can use the following credentials: TestUser@test.net. Note that since the "Order Window" is currently closed, you cannot actually place an order but you can see your "history" of previous orders. This site is functional on Chrome (IE has an issue with the wave file) and includes a responsive design that allows you to order from your phone (but you won't see your history). Future features to be added include getting a list of monthly specials automatically from Cold Stone Creamery (but they don't yet have an API for that). The orders that are entered during an open window get pulled together and FAXED to the local shop for preparation. That's because we work for a faxing software company.

4/30/2015 - Am I Right Calculator

The first project is a twist on the basic calculator built using javascript. However, unlike most standard calculators this one doesn't provide you the answer. Lets say you have 3 numbers to add together: 651 + 345 + 55. In a standard calculator you input them and the calculator provides the answer. That's great if you already know how to add, but what if you are learning how to add? Lets say you "do the math" and come up with 1041. How do you know it's right? If you are my 10 yr old, you pop the numbers into the iPad calculator and it tells you the answer is 1051. Great - so now you have the right answer but maybe not an understanding of where you went wrong. This twisted calculator will instead, allow you to enter the addends, and then allow you a spot to enter your answer. If they match you are correct! If the don't match it will tell you which digits in your answer match the digits in the real answer. This give you the opportunity to fix your math mistake on your own.

I am hoping to use a combination of html5 forms, and css to create a modern layout/design and then of course javascript to do the actual calculations.
